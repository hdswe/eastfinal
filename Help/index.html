The Main Features

This template has modern design and rich functionality. It could be easily modified according to your needs. You can use the variety of menus, create different galleries layouts, portfolios and blog pages and with all this being added your website will look holistically and modern. In Diplomat HTML we have implemented 5 Home Page Layouts and we also have added 4 differnt layouts for gallery and Albums, and 6 differently visualised Blog pages (Blog Classic, Blog Masonry, Layout with captions on images plus excerpt, small thumbnails layout, layout with only captions on images and one more layout with captions under images). The main point is to get you impressed by this template therefore we you have an ability to create various pages with shortcodes and typography elements, moreover you can choose from various animation effects and apply it to that specific element, which will make your website looking dynamical and alluring.

HTML Structure

Despite the variety of different pages, the whole template has unified HTML Structure. All content on the page is being placed in block with id wrapper, which has tag header with id #header, div with id #content, tag footer with id #footer and div with class .bottom-footer.

Subpages

Blog

In Diplomat HTML blog page was added an option to set the post by following types: Post With Gallery, Post With Video, Post Without Image, Post With Image and Post With Blockquote

All posts on Blog page are built in from blocks .entry, .image-entry, .entry-media, .entry-title, .entry-meta, .entry-text

In order to apply specific animation effect to a block with post for tag article with class .entry you need to point that specific animation class. In this example its .opacity. Please note that tag body should have .animated class.

The ability of adding animation to different blocks was implemented with the help of jQuery.appear. And all animation styles described in /scss/_tmm_animations.scss file. If you would like to use some of the animation style for some block, you would need to point the class to which missing word Run to that specific block. For instance, for animation Scale has two classes: .scale and .scaleRun, and .scale should be used while editing. The class .scaleRun appears upon the script usage.

Contact Form

This template comes with working inbuilt contact form.

To make it worked simply open a file js/contfig.js (line 29) and edit there an email address you want to be able receiving all mails from your contact form, then upload all the files to your server.

If you do not want to use captcha in your Contact Form, then plese in line 28, set the 'captcha' value to 'false'.


                    /* ---------------------------------------------------- */
                    /*  Contact Form                                        */
                    /* ---------------------------------------------------- */

                    objContactForm: {
                        captcha: true,                              // Boolean:  (true/false)
                        emailAddress: 'youremail@emaildomain.com'   // Email for contact
                    },

CSS Files and Structure

All css files are located in css/ folder:

  • fontello.css
  • layout.css
  • normalize.css
  • rtl.css
  • styles.css
  • vendor.css

We're using two main CSS files (styles.css and vendor.css) and four addition styles, there are general styles for whole theme and reset styles inside main style.css. Many browsers interpret the default behavior of html elements differently. By using a general reset CSS styles in this file, we can work around this. This file also contains some general styling, such as anchor tag colors, font-sizes, etc. Keep in mind, that these values might be overridden somewhere else in the file.

Please take in mind that this theme uses also .scss file types. The best way to use this theme properly is to use a compiler for such file types. You will find all the necessary documentation how to work with scss file types HERE. Please check below the scss file list, all the files are located in scss/ directory, each of them is responsible for certain part of the website. All the configuration files are included.

  • _functions.scss - Contains all the necessary functions to compile the project
  • _global.scss - Contains global project styles
  • _grid.scss - Contains styles for site grid
  • _third_part_plugins.scss - Contains styles for third part plugins
  • _tmm_animations.scss - Contains styles for elements animations
  • _tmm_base.scss - Contains base styles
  • _tmm_buttons.scss - Contains styles for buttons
  • _tmm_dialog_windows.scss - Contains styles for dialog windows
  • _tmm_events.scss - Contains styles for event pages
  • _tmm_footer.scss - Contains styles for footer
  • _tmm_forms.scss - Contains styles for form elements
  • _tmm_gallery_album.scss - Contains styles for Gallery albums
  • _tmm_header.scss - Contains header styles
  • _tmm_mixins.scss - Contains all mixins
  • _tmm_navigation.scss - Contains styles for navigation
  • _tmm_posts.scss - Contain styles for posts
  • _tmm_settings.scss - Contains all the project settings
  • _tmm_shortcodes.scss - Contains styles for Shortcodes
  • _tmm_styling_options.scss - Contains important Variables for quick website styling
  • _tmm_widgets.scss - Contains widget styles
  • _type.scss - Contains typography styles
  • _vendor_event_calendar.scss - Contains event calendar styles
  • _vendor_jquery_custom_scrollbar.scss - Contain styles for scrollbar
  • _vendor_magnific_popup.scss - Contains styles for magnific popup
  • _vendor_mediaelementplayer.scss - Contains styles for mediaelement player
  • _vendor_owl_carousel.scss - Contains styles for owl carousel slider
  • _vendor_owl_transitions.scss - Contains styles for transitions of owl carousel
  • _vendor_social_widget_counter.scss - Contains social widget styles
  • fontello.scss - Contains all the font icon elements
  • layout.scss - Contains styles for mobile layouts
  • normalize.scss - Contains normalize styles
  • rtl.scss - Contains styles for Right to Left positioned website
  • styles.scss - Contains a list of all necessary styles above
  • vendor.scss - Contains styles for vendor plugins (like mediaelementplayer, custom scrollbar etc.)

If your aim is to only change the styles like colors, fonts, backgrounds etc. you'll find useful to edit the only configuration file (no needs to search each file for a necessary class name to make edition). All the necessary variables you'll find in scss/_tmm_styling_options.scss
